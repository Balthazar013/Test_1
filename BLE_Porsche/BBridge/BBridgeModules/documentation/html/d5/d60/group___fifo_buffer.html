Detailed Description

The goal of this FIFO-Buffer implementation was:

  1. one implementation to store items of any user defined type definition
  2. small footprint
  3. optimized for speed
  4. interrupt safe
  5. Select FIFO-Buffer behavior: Keep oldest or delete oldest
  6. simple to use

int8_t FifoBufferInit ( FifoBufferInstance_t *const  buf,
const FifoBufferMode_t  mode,
const uint16_t  size,
const size_t  sizeOfOneElement 
)

Initialisation of the FIFO-Buffer instance.

If a error occured during the initialisation, the function return the reason.
This circular buffer implementation allocate one element more than specified because:

  1. increase robustness and speed with the modulo operator –> zero check of divisor isn´t necessary
  2. interrupt safe while fully separation of the tail and head index.
    Parameters
    bufConstant pointer of the FIFO-Buffer instance
    modeConfigure the behavior of the FIFO-Buffer if the Buffer is full
    sizeFIFO-Buffer depth
    sizeOfOneElementSize in byte of one element –> use the sizeof-Operator
    Returns
    -3 The FIFO-Buffer is already instantiated
    -2 The FIFO-Buffer behavior mode don´t exist
    -1 The FIFO-Buffer can´t allocate the required memory, fix this error, because you´ll get a memory leak
    1 The FIFO-Buffer is sucessfully allocated and ready to use

int8_t FifoBufferPop ( FifoBufferInstance_t *const  buf,
void *const  value 
)

Pop the oldest element from FIFO-Buffer.

Simplest way to pop elements of type uint16_t from the FIFO-Buffer:

FifoBufferInstance_t myBuffer;
uint16_t popElement;
FifoBufferInit(&myBuffer, FIFO_MODE_LOST_ELEMENT_IF_FULL, 3, sizeof(uint16_t));
FifoBufferPop(&myBuffer, &popElement);

Parameters
bufConstant pointer of the FIFO-Buffer instance
valueGeneric pointer to incomplete type –> universal buffer implementation
Returns
0 The FIFO-Buffer is empty
1 The pop operation sucessfully performed. The caller has exclusive access to the popped value!

Push a new element into the FIFO-Buffer in dependency of the initialized behavior mode.

Simplest way to push elements of type uint16_t into the FIFO-Buffer:

FifoBufferInstance_t myBuffer;
uint16_t pushElement;
FifoBufferInit(&myBuffer, FIFO_MODE_LOST_ELEMENT_IF_FULL, 3, sizeof(uint16_t));
FifoBufferPush(&myBuffer, &pushElement);

Parameters
bufConstant pointer of the FIFO-Buffer instance
valueGeneric pointer to incomplete type –> universal buffer implementation
Returns
0 If the behavior mode is configured to FIFO_MODE_LOST_ELEMENT_IF_FULL, then the buffer is full and the push operation failed
1 The push operation sucessfully performed. If the behavior mode is configured to FIFO_MODE_OVERWRITE_OLDEST_ELEMENT, then the oldest value in the buffer will be overwrite
